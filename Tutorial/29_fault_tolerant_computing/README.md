# Módulo 29: Computación Cuántica Tolerante a Fallos

Este módulo cierra el tutorial conectando la corrección de errores teórica (módulos 09 y 14) con el hardware real (módulos 23-24) y las hojas de ruta de la industria (módulo 28).

## Artículos

1. [Teorema del umbral y escalado subcrítico](01_threshold_theorem.md)
   — Prueba del teorema del umbral, código de superficie a escala, experimento Google Willow.

2. [Magic state distillation y puertas universales](02_magic_state_distillation.md)
   — Por qué el conjunto {H, S, CNOT} no es universal con QEC, estados mágicos, coste de recursos.

3. [Hoja de ruta hacia la tolerancia a fallos](03_hoja_de_ruta_fault_tolerant.md)
   — IBM 2033, Google, Microsoft (qubits topológicos), Quantinuum, cómputo modular.

## Prerequisitos

- Módulo 09 (corrección de errores: códigos de repetición y Shor)
- Módulo 14 (surface codes: síndromes, distancia, umbral)
- Módulo 23 (hardware físico: transmones, coherencia)
- Módulo 28 (benchmarking NISQ, hoja de ruta)

## Relación con el tutorial

Este módulo es el puente entre el mundo NISQ (donde hemos vivido todo el tutorial) y el horizonte de la computación cuántica tolerante a fallos. No requiere implementación en hardware real — los conceptos se ilustran con simulaciones del código de superficie y cálculos de recursos.
